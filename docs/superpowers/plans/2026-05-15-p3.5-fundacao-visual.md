# Plano P3.5 — Fundação visual Wenox OS

> Spec: `docs/superpowers/specs/2026-05-15-wenox-os-roadmap.md` (+ handoff PDF).
> Objetivo: trocar a casca visual (Ionic → Tailwind + shadcn/ui + lucide-react) e montar o
> AppShell do Wenox OS, **sem perder nada do backend, permissões e testes** já em produção.
> NÃO é um dos 11 módulos — é o "chão" onde todos vão morar.

## Resultado esperado

App com a cara do cockpit Wenox OS (sidebar fixa no desktop + drawer no mobile + header com
busca/notificações/tema/perfil), dark default + light, telas atuais (Login, Lista de Clientes,
Detalhe, Form, Usuários, aba Equipe) reescritas no novo design system. Build + testes + e2e
passando. Deploy automático em `app.wenox.com.br` no merge da `main`.

## Fora de escopo (não tocar)

- Backend/PocketBase, regras de permissão, `*Service.ts`, `auth/`, `perms.ts` — preservados.
- Campos novos do cliente (P4), contatos do cliente (P5), acessos (P6).
- Upgrade do react-router (fica na v5; só troca a casca, não o roteador).
- Capacitor (deps ficam; runtime é PWA via Docker/nginx).
- Novos módulos da sidebar ficam **visíveis mas inertes** (link "em breve") — só Clientes e
  Usuários navegam de fato nesta fase.

## Stack a adicionar / remover

**Adicionar:** `tailwindcss` + `@tailwindcss/vite` (Tailwind v4), `lucide-react`,
shadcn/ui (via CLI, base components: button, card, input, label, dropdown-menu, dialog,
sheet, badge, table, sonner/toast, skeleton, avatar, tabs).
**Remover:** `@ionic/react`, `@ionic/react-router`, `src/theme/variables.css`, imports de CSS
do Ionic, componentes `Ion*`. (Capacitor deps mantidas.)

## Design tokens (do doc — obrigatórios)

CSS vars em `src/index.css` mapeadas para o tema shadcn:
roxo `#8B5CF6` (primary), fúcsia `#D946EF` (accent), ciano `#22D3EE` (ring/info),
dark base `#080A16` (background), card `#111327`, texto `#F8FAFC`/`#CBD5E1`,
borda dark `rgba(255,255,255,0.10)`, light base `#F8FAFC` / borda `#E2E8F0`.
Radius alto, sombras suaves com brilho roxo/ciano. Dark = classe padrão no `<html>`;
toggle de tema persiste em `localStorage`.

## Arquitetura de componentes

```
src/
  components/
    layout/
      AppShell.tsx     # grid: Sidebar | (Header + <main> conteúdo)
      Sidebar.tsx      # 11 módulos (lucide icons), item ativo, recolhível mobile (Sheet)
      Header.tsx       # título da área + busca + sino notif + toggle tema + menu perfil
      ThemeProvider.tsx# dark/light + persistência
    ui/                # shadcn components gerados
  pages/ LoginPage.tsx
  clientes/ (telas re-skinned, services intactos)
  usuarios/ ...
```

`App.tsx`: mantém o padrão atual (árvore de rotas trocada por auth — UnauthedApp/AuthedApp),
mas o `IonReactRouter`/`IonRouterOutlet`/`IonTabs` viram `BrowserRouter` (react-router-dom v5)
+ `<AppShell>` envolvendo as `<Route>` autenticadas. **Regra herdada:** nenhuma `<Route>`
pathless dentro do shell — usar `<Route exact>` + `<Redirect>` explícitos (mesma lição do hotfix P3).

## Tarefas (TDD onde aplicável)

1. **Setup Tailwind v4 + alias** — `@tailwindcss/vite` no `vite.config`, `@` → `src` no
   tsconfig/vite, `index.css` com tokens. Build limpo.
2. **shadcn/ui init** — `components.json`, tema dark, gerar os componentes base listados.
3. **ThemeProvider** — dark default, toggle, persistência. Teste: alterna classe no root.
4. **AppShell + Sidebar + Header** — responsivo (sidebar fixa ≥lg; Sheet/drawer <lg).
   Teste: renderiza os 11 itens; item ativo reflete a rota; perfil mostra usuário logado.
5. **Re-skin LoginPage** — card centralizado, inputs nativos (lição jsdom mantida), erro
   visível. Teste de login existente continua verde.
6. **Re-skin Clientes** (Lista com busca/filtros + tabela/cards, Detalhe com segmento
   Info|Equipe, Form criar/editar). Services e e2e de comportamento intactos.
7. **Re-skin Usuários + aba Equipe** — role-aware mantido (`perms.ts`).
8. **Remover Ionic** — desinstalar pacotes, apagar `theme/variables.css`, limpar imports.
   `tsc -b` + `vitest run` verdes.
9. **Atualizar e2e** — seletores que mudaram (`e2e/criar-cliente.mjs`,
   `e2e/abrir-detalhe.mjs`). Rodar local (`E2E_BASE=http://localhost:4173`) e, pós-deploy,
   prod. Limpar registros `E2E Cliente *`.
10. **Build + finishing-a-development-branch** — branch `p3.5-fundacao-visual`, commits
    pequenos, merge `main` → deploy automático. Validar `app.wenox.com.br` no celular e desktop.

## Riscos

- **shadcn/Radix no jsdom**: Dialog/Sheet podem precisar de mocks de `matchMedia`/`ResizeObserver`
  no setup do Vitest. Mitigar no item 2.
- **react-router-dom v5 + AppShell**: cuidado com `<Route>` pathless (lição do hotfix P3).
- **e2e quebra por seletores**: re-skin muda DOM; item 9 ajusta antes de fechar.
- **Tailwind v4** difere da v3 (sem `tailwind.config.js` por padrão, usa `@theme` no CSS) —
  seguir docs v4, não copiar receita v3.
