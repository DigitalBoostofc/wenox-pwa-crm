# P10 — Tarefas

## Escopo travado (decisões de Leonardo, 2026-05-21)

- **Tarefa de projeto** OU **tarefa avulsa interna** (sem projeto = sempre interna da Wenox, sem cliente).
- **Lado responsável:** `Wenox` (responsáveis = usuários da equipe) ou `Cliente` (responsável = um contato do cliente, ex. Pablo/Márcio).
- **Status das tarefas gerenciáveis** em Configurações → Parâmetros (novo tipo `status_tarefa`). Kanban = colunas por status.
- Tarefas aparecem em **3 lugares**: módulo global "Tarefas" na sidebar + aba Tarefas no detalhe do projeto + aba Tarefas no detalhe do cliente.
- **Card do projeto** ganha barra de progresso = etapa atual / total de etapas do pipeline do tipo.
- Cliente terá login no futuro com acesso só ao próprio projeto/empresa — schema e collection rules já preparados; área de aprovação (aprovar / pedir alteração) fica pra fase futura, mas os status já cobrem isso.

## Backend (PocketBase — prod api.wenox.com.br)

1. `opcoes.tipo` += `status_tarefa`. Seeds: **A fazer · Fazendo · Aguardando aprovação · Concluído** (Leonardo ajusta depois).
2. Nova coleção `tarefas`:
   - `nome` (text, obrigatório), `descricao` (text)
   - `projeto` (relation→projetos, opcional, cascadeDelete)
   - `cliente` (relation→clientes, opcional — preenchido a partir do projeto)
   - `lado` (select: `wenox` | `cliente`)
   - `responsaveis` (relation→usuarios, multi) — usado quando lado=wenox
   - `contato` (relation→contatos, single) — usado quando lado=cliente
   - `status` (text — valor de `status_tarefa`)
   - `prazo` (date), `etiquetas` (json), `ordem` (number — Kanban)
   - `created_by`/`updated_by` (relation→usuarios), `created`/`updated` (autodate)
   - Rules list/view escopadas (Owner/Admin/Gestor veem tudo; demais veem tarefas onde são responsáveis ou de projetos onde são responsáveis). CUD = `@request.auth.id != ""`.
3. `historico` e `comentarios`: campo select `entidade` += valor `tarefa`.
